Final Project
Written by: John Louis D. Lagramada
Congrats on finishing the modules! 🎉 It's about time to test your skills with a final project.
Description
The class will be divided into pairs where they will create a simple hardware or pure software project leveraging all the things they learned in this course. In this activity, you will consolidate all the things you learned in this class and create a ROS 2 project just like turtlesim.
Requirements
  1. The ROS 2 system must:
    1. Have at least five (3) nodes
    2. Have at least five (5) messages
    3. Have at least five (5) parameters
    4. Have at least two (2) services
    5. Have at least one (1) action
    6. Have at least one (1) launch file
    7. Use object-oriented programming
    8. (Optionally) Have at least one (1) AI integration
  2. A documentation that includes:
    1. the entire ROS graph
    2. description of the project
    3. design considerations in programming the nodes, services, actions, parameters, and AI integration
    4. ros2doctor results
  3. Recorded messages using rosbag2 for demonstration.
  4. (Optionally) A low-cost robot prototype that must:
    1. Have a name
    2. Not exceed 1000 pesos in cost
    3. Use only an Arduino board for low-level control and a laptop for high-level commands
Criteria
  1. Software Design (50%)
  2. Prototype (30%)
  3. Demonstration (20%)
  4. Additional points are given for exceeding the ROS system requirements and deductions are applied to those who exceed the prototype cost.
Submission
Create a new workspace in your repository called project_ws. Store all your code in that workspace and don't forget to Star our repository for assessment!
